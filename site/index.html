Quickstart

Get started with Latch

Run the installer once in a repository. It adds two workflows and a review policy; Claude stays primary and Codex fallback is explicit, configurable, and off by default.

Claudeprimary review + fix
→
Codexonly after Claude fails
→
Trusted workflowvalidates, posts, stops
One-time setup per repository Node.js 18+, GitHub CLI, Claude access, and optionally a Codex subscription.

  1. Install the workflows

    Run this from the root of the repository you want reviewed.

    $npx github:nishantkumar1292/latch init --codex-fallback

  2. Connect the reviewer

    Install the Claude GitHub App, then add one Actions secret.

    $claude setup-token
    $gh secret set CLAUDE_CODE_OAUTH_TOKEN --app actions
    Use ANTHROPIC_API_KEY instead if you prefer metered API access. Set exactly one secret.

  3. Add Codex fallback optional

    Store a file-backed Codex login as an Actions secret, then enable the repository switch.

    $codex -c 'cli_auth_credentials_store="file"' login
    $gh secret set CODEX_AUTH_JSON < ~/.codex/auth.json
    $gh variable set LATCH_CODEX_FALLBACK --body true
    auth.json is a reusable credential. Latch rejects fork and draft PRs before loading it and removes the runner copy after use. Same-repo branch writers can still influence what Codex reads; enable this only if you accept that boundary.

  4. Check the wiring

    Doctor verifies the files, policy, Git remote, and authentication setup it can inspect safely.

    $npx @nishantkumar1292/latch doctor

Coding-agent setup

Hand the install to your coding agent

Start your coding-agent CLI inside the target repository and paste one prompt. The agent changes files; you retain the credential steps.

  1. Open Claude Code, Codex, Cursor, or another coding-agent CLI in the repository.
  2. Paste the prompt exactly as shown.
  3. Review its commit, then run the human-only commands it returns.

On the pull request

See what Latch changes

Every hop stays visible in GitHub: the provider used, the finding, the repair or disagreement, and the clean re-review.

  1. 01
    Review

    Find what green CI missed

    Claude reviews first. If it cannot finish, Codex posts the same line-level finding with a visible fallback label.

  2. 02
    Fix

    Repair the defect, not the review

    A separate fixer judges the finding, runs checks, commits a repair, and replies with the SHA.

  3. 03
    Stop

    Return the decision to a human

    The re-review is visible, the verdict is non-blocking, and Latch never presses merge.

Safety boundaries

Built to stop

Latch automates the review loop without taking the final repository decision away from you.

Human merge

Neither workflow contains a merge step.

Anti-tamper

The fixer skips changes to .github/workflows/ and .latch/.

Public-repo guard

Fork and draft PRs are rejected before provider credentials load; same-repo branch writers remain inside your trust boundary.

Bounded work

Three unsuccessful fix cycles trigger an author mention and stop.

Your infrastructure

The workflows run in your GitHub Actions. Temporary Codex auth is removed after each fallback attempt.
